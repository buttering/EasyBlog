# 什么是项目
项目就是为创造一件独特的产品、一项服务或者一种结果而进行的临时性努力。
# 项目的特征
- 目的性
- 一次性：不存在完全相同的项目
- 需要资源
- 整体性
- 临时性
- 不确定性：目标的复杂性和可变性
- 多变性：资源需求动态、多变、不确定
- 过程性：随着发展逐渐细化明晰
- 具有发起人
# 项目的组成要素（三维约束）
- 范围
- 时间
- 成本
# 什么是项目管理
以项目为对象的系统管理方法。通过一个临时性的专门的柔性组织，对项目进行高效率的计划、组织、指导和控制，以实现项目全过程的动态管理和项目目标的综合协调与优化。
# 什么是IT项目
包括使用硬件、软件或者网络来创造一件产品、一项服务或者一种结果而进行的一次性努力。
# 什么是软件项目管理
软件项目管理是为了是软件项目能够按照预定的范围、成本、进度、质量顺利完成，而对范围、费用、时间、质量、人力资源、 风险、采购等进行分析和管理的活动。
> 软件项目管理属于IT管理范畴，是IT项目管理的一个核心部分。
# 项目管理的目标（项目成功的标志）
满足利益相关者的需求。
# 软件项目失败的原因
- 软件项目的复杂性
- 缺乏合格的软件项目管理人才
  - 缺乏有效的项目管理是导致软件项目失控的直接原因。
# 活动的3个基本特点
- 目的性：为了达到预期的目的而活动
- 依存性：分工越来越细，依存越来越紧密
- 知识型：在实践与经验中学习，形成知识体系
# 人类活动分为两种类型
- 作业：连续不断、周而复始的活动
- 项目：临时性的、一次性的活动
# 作业和项目的区别（必考）
| 项目             | 作业                   |
| ---------------- | ---------------------- |
| 独一无二         | 重复的                 |
| 有限时间         | 无限时间（相对）       |
| 革命性的改变     | 渐进性改变             |
| 项目之间不均衡   | 均衡                   |
| 目标之间不均衡   | 均衡                   |
| 资源需求多变     | 资源需求稳定           |
| 柔性的组织       | 稳定的组织             |
| 效果性           | 效率性                 |
| 以达到目的为宗旨 | 以完成任务、指标为宗旨 |
| 风险和不确定性   | 经验性                 |
# 项目管理和作业管理
- 项目管理——创新性、追求效果：通过不完全确定的过程、在确定的期限内生产出不完全的产品。
- 作业管理——惯例性、追求效率：作业管理结构需以任务定义为基础来建立，以便进行时间、费用和人力的预算控制，并对技术、风险进行管理。
# 什么是利益相关者
参与项目或受项目影响的个人或组织。
主要项目利益相关者：
- 项目经理
- 客户
- 项目执行组织
- 项目团队成员
- 项目发起方、出资方

# 项目群
多个相互关联项目的集合。
**特征**：
- 相互关联
- 项目相互之间有一定相似性
# 项目群管理
一组相互关联的项目，使用协同方法进行管理以获得收益并进行控制。
# 项目组合管理
对由项目和项目群有机组合的整体所开展的管理。
目的：实现组织战略和愿景。
项目管理和项目组合管理之间最主要的区别在于各自的目标上：项目管理达到战术目标（更具体、时间短），项目组合管理达到战略目标（强调组织的长期目标）。
# 项目经理
## 项目经理的作用
从整体上综合运用各领域的知识以成功实现项目的运作。
## 项目经理的职责
在预算范围内按时优质地领导项目小组完成全部项目工作内容，并使客户满意。
## 项目经理应掌握的技能
- 项目管理知识体系
- 应用领域相关知识、标准和规则
- 项目环境知识
- 一般管理知识和技能
- 软技能/人际关系技能和领导技能
  - 人际关系技能
  - 领导能力
  - 善于倾听
  - 正直、道德、坚定
  - 善于建立信任关系
  - ……
# 项目管理发展阶段
- 传统项目管理：着重在项目实施环节，站在实施方的立场上，分析如何才能更好地完成项目。
- 现代项目管理：注重人的因素，注重顾客，注重柔性管理。追求的是使项目参与方都得到最大的满意及项目目标的综合最优化。基本理念是多赢。
